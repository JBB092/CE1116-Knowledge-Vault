## Definición
Condiciones y validaciones específicas asociadas a una User Story que permiten comprobar si se cumple con lo solicitado por el usuario. Detallan el resultado esperado y aseguran que la funcionalidad sea verificable y satisfactoria para el negocio.

## Ejemplo
User Story:
“Como estudiante, quiero ver mis notas para conocer mi rendimiento.”

Criterios de aceptación:
- El sistema muestra todas las materias cursadas.
- Cada materia incluye su calificación.
- El usuario debe estar autenticado.
- El tiempo de respuesta es menor a 3 segundos.

## Notas
- Reducen ambigüedad en las historias de usuario.
- Sirven como base para pruebas funcionales.
- No describen cómo implementar, sino qué debe cumplirse.
- Deben ser claros, verificables y medibles.
