## Definición
La User Story es un formato para expresar requerimientos desde la perspectiva del usuario, siguiendo la estructura: “Como < tipo de usuario >, quiero < hacer algo >, para < propósito >”. Incluye contexto, criterios de aceptación y resultados esperados para guiar el desarrollo del sistema.

## Notas
- Se enfoca en el valor para el usuario.
- Debe ser clara, breve y comprensible.
- Incluye criterios de aceptación.
- Es la unidad básica de trabajo en Scrum.
